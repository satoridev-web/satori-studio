
# SATORI STUDIO — MASTER DEVELOPMENT ROADMAP (v1.0)


## Product Layering Strategy

### Layer 1 — Open Core
- WordPress
- Beaver Builder Lite (forked as SATORI Studio – Lite)

### Layer 2 — Curated Platform
- SATORI governance
- Stabilisation
- Branding
- Admin UX refinement

### Layer 3 — Commercial Extensions (SSP)
- Advanced modules
- Licensing layer
- Update server
- Advanced forms
- CRM integrations
- Enterprise workflow tools

## Phase 1C — Module/Feature Pruning Audit (Outcome)
- `docs/SATORI-MODULE-AUDIT-PHASE-1C.md` created with module and admin feature inventory plus recommendations.
- Legacy Beaver Builder upsell surfaces (upgrade/license screens, Themer/Assistant promos, builder upgrade CTAs) are soft-hidden behind new SATORI feature flags.
- No content modules were removed or renamed; candidates for hard removal are deferred to later phases.

## Phase 1 — Core Acquisition & Forking (Weeks 1–3)
### Goal: Create a stable, white-labelled foundation based on Beaver Builder Lite.
- Obtain and freeze BB Lite release.
- Create private GitHub repo `satoridev-web/satori-studio-core`.
- Rename plugin, text domain, namespaces.
- Replace branding, icons, UI colours.
- Verify GPL compliance.
- Remove unused modules/features.

## Phase 2 — SATORI Studio Foundation Enhancements (Weeks 3–6)
### Goal: Improve base experience + prepare for new modules.
- Implement SATORI design system.
- Builder UI refresh (side panel, fields, toggles).
- Global Settings Panel (colors, typography, spacing).
- Section Presets / Patterns Library.

## Phase 3 — SATORI Studio Base Modules (Weeks 6–12)
### Goal: Replace UABB & PPBB essentials with SATORI modules.
- Essential modules (Heading, Button, Image Box, Accordions, Tabs, Grids).
- Enhanced media modules (Gallery, Carousel, Video Popup).
- Standardized module framework.
- ACF dynamic data, animations, responsive controls.

## Phase 4 — SATORI Studio Pro (Weeks 12–20)
### Goal: Build the premium offering.
- Advanced slider modules, hotspots, mega menu, Lottie.
- Global style features, custom breakpoints, role access.
- Licensing API & update service.

## Phase 5 — SATORI Suite Add-On Integrations (Weeks 20–32)
### Goal: Convert SATORI Suite plugins into Studio add-ons.
- Events Add-On (grids, calendars, meta blocks).
- Forms Add-On (multi-step, popup forms, field styling).
- CRM Add-On (user dashboard, conditionals, membership CTAs).
- Audit Add-On (admin-only modules, scorecards).
- Tools Add-On (animations, spacing, typography manager).

## Phase 6 — SATORI Studio Theme (Weeks 32–44)
### Goal: Build a theme optimized for SATORI Studio.
- Block theme/hybrid approach.
- Fluid responsive framework.
- Header/footer builder.
- WooCommerce template optional support.

## Phase 7 — Marketplace & Distribution (Weeks 44+)
### Goal: Commercial platform for updates, licenses, templates.
- Template cloud library.
- Add-on marketplace.
- Licensing dashboard.
- Developer docs and KB.
